1. what is state?

-> session : client --> server interaction in one go. :: the conversation
-> protocol : the protocol used in the req-res cycle.

--> types of protocols : 
-state : the related info about the session or the transaction

-stateful protocol: 
-----: protocol which stores state of the session; requires server to save the status and session info, eg : ftp

-stateless protocol:
-----: protocol which doesn't saves any states or any related info of the session; doesnot require the server to retain the server information or any related info. , eg : http
